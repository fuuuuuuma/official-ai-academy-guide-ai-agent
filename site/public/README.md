## プレゼントサイトでできること

1. 5社の公式アカデミーと推奨講座を開く。
2. 5社の学習内容をまとめた参照ノートを、普段使うAIへ一括コピーまたはファイル添付する。
3. OpenAI・Anthropic・Google・Cursor・Manusごとに、講座内容から作った20本のプロンプトを個別／20本単位／100本一括でコピーする。
4. MarkdownまたはJSONを保存して、自分のプロジェクト資料として再利用する。

> 「AIへ取り込む」はモデルを恒久的に再学習させる意味ではなく、現在の会話・プロジェクトで参照させるコンテキストとして渡す意味です。

## AIツール別・一括取り込み手順

### 共通手順

1. [academy-knowledge.md](./academy-knowledge.md)を保存するか、プレゼントサイトの「AI学習用テキストをコピー」を押す。
2. 普段使うAIの新しい会話・プロジェクトへ、ファイルを添付または全文を貼り付ける。
3. 「この学習ノートを現在の会話の基準資料として使ってください。事実・推奨・推測を分けてください」と伝える。
4. 自分の仕事、利用する資料、期待する成果物、完了条件、人が確認する箇所を追加する。
5. [prompts.md](./prompts.md)から目的に近いプロンプトを選び、角括弧の部分を自分の内容へ置き換える。

この方法は、AIモデルそのものを恒久的に再学習させる操作ではありません。会話やプロジェクト内で、回答時に参照する基準資料を渡す方法です。新しい会話へ移る場合は、必要に応じて同じファイルをもう一度添付してください。

### ChatGPT / OpenAIで使う

ChatGPTの会話またはプロジェクトへacademy-knowledge.mdを添付し、OpenAI Academyの「文脈・目標・出力形式・境界・レビュー」の順番で仕事を設計させます。最初に「業務候補を棚卸しする」、次に「完了条件を定義する」、最後に「レビュー担当を設計する」を使うと、単発の質問から再利用できる工程へ進めやすくなります。外部送信や公開を含む場合は「承認境界を置く」を先に実行してください。

### Claude / Claude Codeで使う

Claudeの会話へ添付する場合は学習ノートとして参照させ、Claude Codeで使う場合はリポジトリ内のdocsなどへ保存して@参照します。「リポジトリ地図を作る」から始め、CLAUDE.md、Plan、最小差分、テスト、最終検証の順で使います。MCP・Hooks・GitHubなど外部操作を伴う項目は、読み取りと書き込みの権限を分け、変更前に計画を確認してください。

### Gemini / Google AIで使う

Geminiの会話へ学習ノートを渡し、生成AI・LLM・責任あるAIの基礎を前提に、システム指示、安全設定、会話状態、関数呼び出しなどを設計します。実装へ進む場合はGen AI SDKの利用環境・認証情報・利用条件を公式ページで確認し、キーを本文やGitへ書かないでください。

### Cursorで使う

academy-knowledge.mdとprompts.mdをリポジトリのdocsへ置き、対象タスクに必要なファイルと一緒に参照します。「コードベースを把握する」「必要なコンテキストを選ぶ」「機能実装の計画を作る」の順に進め、変更後は差分レビュー・テスト・最終QAを行います。AIが存在しないAPIやファイルを仮定しないよう、根拠パスを求めてください。

### Manusで使う

Manusへ学習ノートを添付し、調査・ブラウザ操作・アプリ操作・成果物・QCへタスクを分解させます。送信・公開・購入・削除の前で停止する承認チェックポイントを置き、リサーチでは一次情報と確認日を残します。文書・スライド・表計算・Webを同時に作る場合は、共通の事実と出典を一つの基準資料へ集約します。

## この資料の方針

業務設計・調査・コンテンツ制作・実行型AI活用の観点を最初に置きます。 ただし、特定の製品を推す比較資料ではなく、5社を一社ずつ学習の入口として解説します。

この資料は「OpenAIとAnthropicの比較」ではありません。各サービスに無料または無料で始められる公式学習リソースがあり、目的・前提・進め方が異なります。したがって、動画も資料も **1社を説明し切ってから次へ進む** 構成です。各章は次の順番を守ります。

1. 公式アカデミーのリンクを最初に表示する。
2. 実際の公式画面を見せる。
3. 具体的な推奨講座の直リンクを出す。
4. そこで学ぶ内容を、日本語で噛み砕く。
5. 向く人・前提・最初の一歩・受講前の注意を示す。

英語ページについては、公開されているコース概要、目次、対象者、必要環境を読み取り、日本語で要約しています。ログイン後にしか見られない動画本文・字幕・配布資料を、見えていないのに「文字起こしした」とは記載していません。動画がある講座は、公開ページで確認できる講義数・時間・カリキュラムを明示し、受講後に確認すべき実践ポイントへ訳しています。

## 受講前に確認すること

- 「無料」はサービス全体の全コンテンツが無条件で無料、という意味ではありません。アカウント、無料クレジット、地域、機能アクセス、受講枠が関わる場合があります。
- Google Skillsはコースによって表示・クレジット条件が異なります。資料では「No cost」表示を確認できるハンズオンを補足していますが、開始画面の条件を確認してください。
- Manusは日本語の基礎コース画面にサインインが表示されます。利用枠、料金、機能条件をアカウント側で確認してから進めてください。
- OpenAI Academy、Anthropic Academy、Cursor Learnも、UI・カリキュラム・受講条件は更新され得ます。公開直前と受講直前にリンクを再確認してください。

## 1. OpenAI Academy

### 学べること

- 文脈を渡す設計
- 出力形式と境界の決め方
- レビューと改善の回し方
- 繰り返し業務をワークフロー化する考え方

### 公開情報から読める講座の構成

公式ページでは約90分・セルフペース・中級者向けとして案内。AI Foundations と Applied AI Foundations の次に、普段の仕事を一つ持ち込んで学ぶ構成です。

### おすすめする人と、開始後の行動

ChatGPTを単発利用で終わらせず、毎週の調査・資料作成・確認作業を「設計して任せる」側に進みたい人。

開始したら、ホームの AI Foundations を確認し、次にこの講座で「毎週くり返す1業務」を題材にする。

**受講前の注意：** 受講にはChatGPTアカウントと、エージェント機能にアクセスできる環境が案内されています。

- MCP・GitHub連携・Hooksによる自動化

### 公開情報から読める講座の構成

**受講前の注意：** 公式案内で示される前提知識はCLI操作とGitの基礎です。教材閲覧にはSkilljarアカウントを使います。Claude Codeを実際に動かす際のアカウント・利用環境は別途確認してください。

## 動画撮影の導線
