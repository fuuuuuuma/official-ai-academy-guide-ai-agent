# 公式AIアカデミー学習ノート｜AIエージェントチャンネル

## AIに渡すときの指示

以下を「公式AIアカデミー学習ノート」として、この会話またはプロジェクト内の参照情報にしてください。回答では、事実・推奨・推測を分け、更新され得る条件は公式URLの確認を促してください。このノートはモデルの恒久的な再学習ではなく、現在の会話・プロジェクトで使う参照コンテキストです。

- 5社を優劣だけで比較せず、それぞれの学習目的に合わせて使い分ける
- 講座内容から外れる提案は「追加提案」と明記する
- 外部送信・公開・削除・上書き・課金は、人の確認前に実行しない

## 1. OpenAI Academy（ChatGPT / OpenAI）

- 公式入口: https://academy.openai.com/
- 推奨講座: Agents and Workflows — https://academy.openai.com/public/courses/agents-and-workflows-bieml
- 講座から抽出した軸: 文脈、目標、出力形式、境界、レビュー、反復改善を備えた再現可能なワークフロー設計
- 公開情報: 公開ページでは約90分・セルフペース・中級者向け / AI FoundationsとApplied AI Foundationsの次に進む位置づけ / 実際の繰り返し業務を題材に、エージェントの文脈・出力・境界・レビューを設計
- 学べること:
  - 単発の依頼を再利用できるワークフローへ分解する
  - 必要な文脈と期待する出力形式を先に定義する
  - 人が承認する境界と品質レビューを設ける
  - 失敗結果を材料に指示と工程を改善する
- おすすめする人: ChatGPTを単発利用で終わらせず、調査・資料作成・定型業務を再現可能な手順へ変えたい人
- 注意: ChatGPTアカウントと利用可能な機能は環境により異なります。重要な判断と外部操作は人が確認してください。

## 2. Anthropic Academy（Claude / Claude Code）

- 公式入口: https://www.anthropic.com/learn
- 推奨講座: Claude Code in Action — https://anthropic.skilljar.com/claude-code-in-action
- 講座から抽出した軸: リポジトリ理解、CLAUDE.md、コンテキスト参照、Plan、カスタムコマンド、MCP、GitHub、Hooks、SDK
- 公開情報: 公式案内では無料・15講義・動画約1時間・クイズ1回・修了証 / 前提知識はCLI操作とGitの基礎。教材閲覧にはSkilljarアカウントを使用 / セットアップからMCP・GitHub・Hooks・SDKまでを一続きで扱う
- 学べること:
  - ファイルとコマンドを使ってコードベースを理解する
  - CLAUDE.md・/init・@参照でコンテキストを管理する
  - Planと小さな差分で安全に変更する
  - MCP・Hooks・SDKで反復作業を自動化する
- おすすめする人: Claude Codeをチャットではなく、自分のリポジトリで計画・変更・検証する開発環境として使いたい人
- 注意: 変更前に計画と対象ファイルを確認し、削除・公開・外部送信などの操作には承認点を置いてください。

## 4. Cursor Learn（Cursor）

- 公式入口: https://cursor.com/ja/learn
- 推奨講座: Working with agents — https://cursor.com/learn/working-with-agents
- 講座から抽出した軸: モデルの仕組み、幻覚、トークン、コンテキスト、ツール呼び出し、エージェント、コード理解、機能実装、バグ修正、レビュー、テスト
- 公開情報: 公式LearnはAI basicsとCoding agentsの教材群で構成 / Working with agentsからコードベース理解・実装・修正・レビューへ進める / AIの限界と検証を前提にコーディングエージェントを扱う
- 学べること:
  - コードベースに必要な文脈を選んで渡す
  - 変更前に対象範囲と検証方法を決める
  - 機能実装・バグ修正・リファクタリングを小さく進める
  - 差分・テスト・セキュリティを人が確認する
- おすすめする人: AIエディタを補完だけで終わらせず、既存コードを理解して変更を検証する共同作業者として使いたい開発者
- 注意: 生成されたコードをそのまま採用せず、差分・テスト・依存関係・権限への影響を確認してください。
